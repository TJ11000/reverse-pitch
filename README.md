# Project Reverse Pitch

> **AIのドリフトは直さない。受容して、被ダメージを最小化する。**

---

## このプロジェクトは何か

長セッションで AI（Claude / Gemini / GPT 系）の記憶劣化・前提崩壊・捏造が発生する問題を、「直す」のではなく「**起きる前提で運用する**」アプローチで扱う設計プロジェクトです。

本リポジトリは、そのプロジェクトの**思想・主張・発案記録**を公開する公的な足あとです。実装手順・運用詳細・販売物の中身は含まれていません。

## 立場と方針

- 本プロジェクトは特定 AI ベンダーへの批判・対立を目的としません
- 「ドリフトは仕様に近い構造特性である」という観察に立脚し、**運用設計でそれを補う**ことを志向します
- 引用・参照・派生利用は歓迎します。条件は [LICENSE](./LICENSE) を参照

---

## 何が含まれていないか

意図的に以下を非公開とします：

- 実装手順・商品設計・UX・運用テンプレート
- AIベンダーとの対話プロトコル・交渉条件
- ドリフト検知の具体メカニズム・尋問技法・再現手順
- 個別の運用ノウハウ・チェックリスト・内部ログ

中身を出さないこと自体が、本プロジェクトの設計の一部です。
